# 提案：文章库增加类型与语言筛选

## 背景和目标

文章库页面目前可以按关键词和状态查询，但运营人员无法直接筛选文章类型或语言。当文章积累后，查找指定类型的中文或英文内容需要逐页浏览。

本次优化让运营人员可以组合关键词、状态、文章类型和语言快速定位文章。

## 变更内容

- 在文章库查询区增加“文章类型”和“语言”两个可清空的下拉筛选项。
- 查询、重置和分页时，保留并正确提交已选择的筛选条件。
- 复用现有文章列表接口的 `type` 与 `language` 查询参数，不新增接口、数据库字段或状态。

## 范围与不做事项

本次只调整文章库列表的筛选交互。

不做事项：

- 不新增排序、保存筛选条件、批量操作或全文检索。
- 不修改文章生成、文章编辑、平台稿和发布任务流程。
- 不修改后端接口、数据库或既有前后端契约。

## 验收检查

- 运营人员可单独或组合选择文章类型和语言后查询。
- 查询结果仅包含同时满足所选条件的文章。
- 点击重置后，所有筛选条件清空并重新加载第一页。
- 翻页时保留当前筛选条件。

## 能力

### 新增能力

- `article-library-filters`：文章库按文章类型和语言筛选文章的界面能力。

### 已修改能力

- 无。当前项目尚未建立可复用的文章库 OpenSpec 规格；本次新增首个相关规格。

## 影响

- 前端：`frontend/src/pages/Article/index.tsx` 使用已有的筛选选项与查询类型。
- 后端：不改代码；现有 `GET /api/articles` 已支持 `type` 与 `language` 参数。
- API、数据库、安全、外部平台和发布状态机：无影响。
