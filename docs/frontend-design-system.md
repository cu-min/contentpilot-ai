# AI内容营销系统前端设计系统

## 1. 设计风格定位

AI内容营销系统是公司内部运营工具，前端采用科技极简的后台中台风格。界面应突出清晰层级、稳定信息密度、专业运营感和轻量数据感，避免营销官网式的大面积装饰。

关键词：科技感、简洁、留白、清晰层级、蓝紫色点缀、卡片式布局、轻量阴影、圆角、数据感、专业运营工具。

## 2. 主色与辅助色建议

- 主色：Ant Design 默认蓝 `#1677ff`，用于主按钮、关键链接、选中态。
- 科技辅助色：紫蓝 `#6d5dfc`，用于页面顶部细节、说明区渐变或状态点缀。
- 背景色：`#f5f7fb`，用于后台页面整体背景。
- 卡片色：`#ffffff`，用于表单、表格和内容承载。
- 边框色：`#e5e7eb`，用于分割线和卡片边界。
- 正文色：`#1f2937`，次级文字 `#667085`。
- 状态色优先使用 Ant Design 内置 `success`、`warning`、`error`、`processing`。

## 3. 字体和字号层级

- 字体使用系统无衬线字体：`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`。
- 页面标题：24px，600-700 字重。
- 区块标题：18px，600 字重。
- 表单标签和正文：14px。
- 辅助说明：13px，使用次级文字颜色。
- 不使用随视口变化的字体大小，避免后台页面层级不稳定。

## 4. 页面布局规则

- 后台页面使用左侧菜单 + 顶部栏 + 内容区布局。
- 内容区保持 24px 内边距，复杂页面可拆成顶部说明区和主体卡片区。
- 表单页宽度适中，建议主体表单最大宽度 960px，避免整屏拉伸导致阅读困难。
- 页面顶部可以使用轻量渐变或细线装饰，但不要使用复杂背景图、装饰球或大面积强色块。
- 优先复用 `PageContainer` 和 `SectionCard`，让页面标题、说明、卡片间距保持一致。

## 5. 表单设计规则

- 使用 Ant Design `Form`，标签左侧对齐或顶部对齐均可，后台配置页优先垂直布局。
- 必填项清晰标记，错误提示使用 Ant Design 表单校验。
- 长文本字段使用 `TextArea`，建议 3-5 行。
- 保存按钮放在表单底部，使用主按钮；加载时显示 `loading` 状态。
- 表单字段顺序应贴合用户理解：基础信息、产品能力、目标用户、优势、语气和禁用词。

## 6. 卡片设计规则

- 卡片用于承载独立表单、表格或信息区，圆角保持 8px 以内。
- 卡片边框使用浅边框，阴影轻量克制。
- 不嵌套卡片；需要分区时使用标题、分割线或内部网格。
- 卡片留白要足够，常规内边距 20-24px。

## 7. 按钮设计规则

- 主要提交动作使用 `type="primary"`。
- 次要动作使用默认按钮。
- 危险动作才使用 `danger`，并需要确认或明确文案。
- 按钮文案保持短句，如“保存配置”“新增用户”“退出登录”。
- 图标按钮后续优先使用项目统一图标库或 Ant Design 图标，不临时拼 SVG。

## 8. 表格设计规则

- 表格用于管理类列表，保持列名清晰、操作列靠右。
- 状态字段用 Tag 或 Switch 表示，不使用纯文本堆叠。
- 分页使用 Ant Design 默认分页。
- 搜索条件不要过度复杂，MVP 阶段保持简单关键词即可。

## 9. 状态提示设计规则

- 成功、失败、加载使用 Ant Design `message`、`Alert`、`Spin`。
- 表单保存成功后显示短提示，不跳转页面。
- 接口失败显示后端返回 message；401 统一清登录态并跳转登录页。
- 空状态使用简短说明，避免长篇解释。
- 发布任务页只展示当前状态机中的 `DRAFT`、`PENDING`、`RUNNING`、`SUCCESS`、`FAILED`、`CANCELLED`。
- `FAILED` 任务必须支持换行展示失败原因；`SUCCESS` 任务只在存在发布链接时显示“查看文章”。

## 10. 后续页面开发注意事项

- 严格沿用产品设计文档阶段边界，不提前实现后续复杂能力。
- 新页面优先复用 `src/components/PageContainer` 和 `src/components/SectionCard`。
- API 请求放在 `src/api`，类型放在 `src/types`，不要散落在页面组件中。
- 避免写死真实密钥、平台 Cookie、AI Key 或个人路径。
- 继续保持 ADMIN 与 OPERATOR 的权限边界：用户管理仅 ADMIN 可见，其他业务功能两种角色均可使用。
- 浏览器自动化相关文案只描述当前状态和人工处理动作，不承诺绕过登录、验证码或风控。
