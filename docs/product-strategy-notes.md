# 产品策略笔记

> 写于 2026-07-21。记录关于 ContentPilot AI 产品定位、竞争壁垒和后续方向的讨论。

## 项目背景

ContentPilot AI 最初是公司内部练手项目（实习期间开发），从产品设计文档出发，完成了 Spring Boot + React 全栈实现、Docker 部署和腾讯云上线。

核心链路：产品配置 → AI 生成文章 → 四平台内容适配 → 发布任务 → 草稿/编辑器准备 → 人工发布。

## 核心判断

### 这个项目证明了什么

- 独立完成全栈项目从设计到上线的完整链路
- 工程素养：分层架构、安全措施、CI/CD、测试覆盖
- 接触了 AI 应用开发、浏览器自动化、平台 API 集成

### 这个产品目前缺什么

**缺真实用户验证。** 所有模块都是按照产品文档做的，没人真正用过。不知道哪些功能有用、哪些是多余的。

**缺"回头钩"。** 用户生成一篇文章就跑了，没有理由回来。缺少数据回收和效果对比闭环。

## 关于竞争壁垒

### "Prompt 改写"没有壁垒

当前平台改写逻辑本质是：`平台风格描述 + 原文 → Prompt → DeepSeek → 改写结果`。任何人复制你的 prompt 就能得到类似效果。技术上零壁垒。

### 壁垒应该建在「改写知识库」上

真正的护城河不是改写技术，而是跨平台效果数据：

```
改写 → 发布 → 数据回收 → 效果对比 → 反馈到改写策略 → 下次改写更精准
```

每个平台维护一个**可量化、可迭代的改写规则集**，基于真实发布数据持续更新：
- 标题策略（字数、结构、关键词偏好）
- 正文结构（段落长度、技术深度、营销浓度）
- 发布窗口（最佳发布时间）
- 踩坑记录（限流规则、禁用词、外链策略）

AI 只是执行引擎，知识库才是壁垒。这套知识库积累到一定数据量后，后来者很难追赶。

## 多平台扩展方向

当前支持：微信公众号、知乎、CSDN、掘金（均为长文平台）。

扩展方向需注意不同平台的内容形态差异：

| 平台 | 内容形态 | 改写复杂度 |
|---|---|---|
| 微信公众号 | 长文 | 低（已支持） |
| 知乎 | 问答/长文 | 低（已支持） |
| CSDN | 技术博文 | 低（已支持） |
| 掘金 | 轻量技术分享 | 低（已支持） |
| 小红书 | 短笔记 + 配图 + 标签 | 中（需生成配图方案） |
| 微博 | 短文案 140 字 + 话题 | 低（但生命周期极短） |
| 抖音 | 短视频脚本 | 高（脚本分镜，非文章） |
| B站 | 长视频脚本/专栏 | 高（不同形态） |

扩展平台不只是"加一个 prompt"，需要适配不同媒介格式。

## 下一步优先级

### 不要做的事

- ❌ 现在投流推广（没有验证留存率，花钱放大失败）
- ❌ 关门打磨功能（不知道用户卡在哪，猜着改没用）
- ❌ 继续加新模块而不验证旧模块

### 应该做的事

1. **找 5 个真实用户试用以**（不是"帮看看"的朋友，是真的做内容分发的人）
2. **观察他们怎么用**，找到最卡的环节
3. **只修最痛的一个点**
4. **验证"有人愿意回来用第二次"**
5. **做出一个成功案例**再考虑推广

PMF 的最低验证标准：**在没有金钱激励的情况下，5-10 个人自发、持续地使用，频率超过每周一次。**

达到这个之前，不要推广。
